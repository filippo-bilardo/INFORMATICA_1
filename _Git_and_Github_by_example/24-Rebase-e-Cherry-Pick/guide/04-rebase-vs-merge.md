## üîÑ Confronto Visuale

### Merge: Preserva la Cronologia

```
Prima del merge:
    A---B---C feature
   /
  D---E---F main

Dopo git merge feature:
    A---B---C feature
   /         \

### Rebase: Cronologia Lineare

```

    A---B---C feature

  D---E---F main

              A'--B'--C' feature
             /
  D---E---F main

## üìä Confronto Dettagliato

| Aspetto | Merge | Rebase |

| **Cronologia** | Ramificata, preserva contesto | Lineare, semplificata |

| **Collaborazione** | Sicuro per branch condivisi | Solo per branch locali |
| **Tracciabilit√†** | Storia completa | Storia semplificata |
| **Conflitti** | Una volta sola | Potenzialmente multipli |
| **Reversibilit√†** | Facile da annullare | Pi√π complesso |
| **Complessit√†** | Semplice | Richiede esperienza |

## üéØ Quando Usare Merge

### ‚úÖ Scenari Ideali per Merge

### üèÜ Vantaggi del Merge

- **Sicurezza**: Non modifica commit esistenti
- **Tracciabilit√†**: Preserva quando e come √® stata integrata una feature

### ‚ö†Ô∏è Svantaggi del Merge

- **Cronologia complessa**: Pu√≤ diventare difficile da leggere
- **Merge commit**: Commit aggiuntivi che "inquinano" la storia
- **Debugging difficile**: Pi√π difficile seguire il flusso logico
- **Bisect problematico**: `git bisect` meno efficace

## üîÑ Quando Usare Rebase

### ‚úÖ Scenari Ideali per Rebase

1. **Aggiornamento Branch Feature**
   ```bash
   # Incorpora ultime modifiche di main
   git checkout feature/new-ui
   git rebase main
   ```

2. **Pulizia Pre-Merge**
   ```bash
   # Pulisci la cronologia prima del merge finale
   git rebase -i HEAD~5
   git checkout main
   git merge feature/clean-history
   ```

3. **Sincronizzazione Fork**
   ```bash
   # Aggiorna fork con upstream
   git rebase upstream/main
   ```

4. **Branch Personali**
   ```bash
   # Riorganizza il proprio lavoro
   git rebase -i main
   ```

### üèÜ Vantaggi del Rebase

- **Cronologia pulita**: Storia lineare e leggibile
- **Debugging efficace**: Facile seguire il flusso logico
- **Bisect efficiente**: `git bisect` molto pi√π efficace
- **Commit atomici**: Ogni commit rappresenta un'unit√† logica completa
- **Review migliori**: Code review pi√π facili da seguire

### ‚ö†Ô∏è Svantaggi del Rebase

- **Modifica cronologia**: Cambia hash dei commit esistenti
- **Conflitti multipli**: Possibili conflitti su ogni commit
- **Complessit√†**: Richiede maggiore esperienza
- **Perdita contesto**: Non preserva il timeline di sviluppo parallelo
- **Rischio collaborazione**: Pericoloso su branch condivisi
